Welcome to the COMP 215 Pretty Pictures Breeder! The controls here are simple.

Starting the breeder

  • The TEST 1 button below will request the standard test generation defined in the project description using the /test/ route. Implementing this is your goal for week 1. In week 2, you'll add more tests.
  • The ↺ button sends a reset request to the PrettyPictures server using the /reset/ route, then loads the first generation.
  • The ► button is only active if the PrettyPictures server responded to the /client-init/ request route and reported extant generations.

Using the breeder

  • The RESET button sends a reset request to the server like the ↺ button.
  • The PREV and NEXT buttons change the current generation.
  • The text box tracks the current generation being displayed and the maximum generation that's been bred. PREV and NEXT will not navigate beyond the maximum generation.
  • The population of a generation will appear in this area. Click an image to select it for breeding.
  • The 🔍 button on an image will display the image at a larger resolution in an overlay. Click on the image to open it in a new tab, or click anywhere else or press ESC to close it.
  • The ⚤ button on an image will open its genotype in a new tab using the /string/ route.
  • The DNA button breeds a new generation from the selected images if there are at least two, using the /breed/ route.
  • If you've just bred a generation and haven't selected anything, pressing BREED again will re-breed the latest generation.
  • Note: Breeding overwrites any generations that used to come after the newly bred generation. Depending on your implementation, this is probably irreversible!
  • When you're ready to begin, click a button below.
